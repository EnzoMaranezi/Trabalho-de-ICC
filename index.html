Como usar o GitHub e seus códigos

  1. Para começar, você deve acessar o link https://github.com e criar sua conta com suas informações.
  2. Após o login, você deve criar um repositório no GitHub.
  3. Com o URL do repositório criado, você pode colar o mesmo no terminal do seu computador ou usar o GitBash para utilizar os comandos.
  4. Ao entrar no diretório do projeto no terminal ou no GitBash, o comando git init vai inicializar um repositório LOCAL.
  5. Com o comando git remote add origin URL_do_repositório, a conexão do repositório local com o repositório do GitHub será feita.
  6. Os comandos git add, git commit, git push serão utilizados para adicionar ou modificar repositórios no GitHub. -git add: adicionar conteúdo do arquivo ao índice. git commit: é usado para promover alterações no projeto. git push: permite que você envie (ou em tradução literal, empurre) os commits de sua branch e repositório Git local para o seu repositório remoto.
  7. Caso você queira copiar um repositório Git de outra pessoa, isso é possível com o comando git clone, copiando o URL do repositório remoto. Com o GitBash, use o comando git clone URL_do_repositório.
  8. Branch é um tipo de ramificação de um código, de modo que seja possível testar funções sem comprometer o mesmo. Para criar uma branch, basta usar o comando git branch nome_da_branch.
  9. Para alterar entre as branches criadas, basta usar o comando git checkout nome_da_branch, só sendo possível alterar entre repositórios existentes no sistema local.
  10. Para verificar as condições do diretório, basta usar o comando git status, permitindo a visualização.
  11. O comando git diff permite uma comparação entre o diretório de trabalho e o último commit.
  12. Para baixar o conteúdo de um repositório online em um repositório local, usa-se o comando git pull nome_do_repositorio.
  13. O comando git revert é utilizado para desfazer um commit.
  14. Também é possível unir duas branches da sua escolha. Estando na branch que você quer que a outra seja aderida, use o comando git merge nome_da_branch.
  15. O comando git stash "arquiva" alterações que você tenha feito durante um período de tempo.
  16. O git rm é considerado o oposto do git add, sendo utilizado para remover arquivos do repositório do Git.
  17. O comando git config é utilizado para definir valores, como nome e email.
  18. O comando git reset é usado para ajustar ou desfazer alterações em um repositório Git. Ele é comumente usado para desfazer commits, redefinindo o estado do repositório para um commit anterior.
  19. O comando git remote gerencia os repositórios remotos que você monitora.
  20. "O comando git fetch baixa commit e arquivos de repositórios remotos para o seu local.
  21. "O comando git show é utilizado para exibir informações,como commits e marcações.
  22. O comando git help pode te ajudar caso voce queira ajuda.
  23. O comando git rebase serve para juntar os commits em cima de outra base.
  24. O comando git blame serve para você ver todos os commits que estão mexendo no arquivo.
  25. Para criar marcações,existe o comando git tag.
